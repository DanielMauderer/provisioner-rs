# Feature 6 — Storage format & impl

## Goal

Persist the configured struct durably behind the already-implemented `Storage`
trait, and restore it on the next boot so the portal can be skipped. The
serialization format is generated by the macro (`to_bytes`/`from_bytes`); the
ESP32-C3 backend writes to a raw flash partition.

The `Storage` trait is already defined in `storage.rs`:

```rust
pub trait Storage {
    type Error;
    fn load(&mut self, buf: &mut [u8]) -> Result<usize, Self::Error>;
    fn store(&mut self, data: &[u8]) -> Result<(), Self::Error>;
}
```

## Subtasks

- [M1] Define a versioned byte serialization format for the config struct, with
  a magic marker + version header so stale/foreign data is rejected. The macro
  emits `to_bytes`/`from_bytes` matching this layout (see Feature 1).
- [M1] Implement `Storage` for ESP32-C3 using `sequential-storage` +
  `esp-storage` against a raw flash partition (explicitly **not** ESP-IDF NVS).
- [M1] A validity check (magic + version + length) used by the platform
  boot-decision helper (Feature 5) to decide whether to skip the portal.
- [M2] Format versioning / migration story, plus corruption handling (CRC or
  checksum, fall back to re-provisioning on mismatch).

## Public surface / signatures

- Runtime: the existing `Storage` trait + an esp32c3 impl (constructed from the
  flash partition).
- Macro-generated, per config type:
  `fn to_bytes(&self, buf: &mut [u8]) -> Result<usize, ParseError>` and
  `fn from_bytes(buf: &[u8]) -> Result<Self, ParseError>`.

## Test setup

- Host tests for serialize → deserialize round-trip using an in-memory `Storage`
  mock (a `Vec`/array-backed impl), including the validity/version check.
- Document a flash-backed round-trip test on hardware (write, reboot, read).

## Open questions / risks

- Exact wire format: keep it simple and deterministic (length-prefixed fields)
  so the macro can generate it mechanically.
- Buffer sizing: the serialized form must fit a single page/region; bound it and
  surface an error if a config is too large.
- Coordinating the format version constant between the macro output and the
  validity check.
